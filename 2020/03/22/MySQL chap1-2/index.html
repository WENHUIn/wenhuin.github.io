Basic 基础

一 检索数据

  • SELECT column name
    FROM table

    1. 多条SQL要用分号；区分

    2. SQL不区分大小写

    3. SQL语句多行写，便于阅读与调试

  • 选择多个列

    SELECT column1，column2

    FROM table

  • 对于行，只返回不同的值

    SELECT DISTINCT column

    FROM table

  • 限制行数

    SELECT *
    FROM table

    LIMIT number

    or LIMIT 5,5 从第5行开始再往后数5行

二 排序检索数据

  • 按多个列排序

    SELECT column1,column2,column3

    FROM table

    ORDER BY column1,column2

    ++eg. order by price, name++
  • 降序与升序(default)

    SELECT *

    FROM table

    ORDER BY price DESC
  • 组合

    SELECT *

    FORM table

    ORDER BY column DESC

    LIMIT num

三 过滤数据（filter condition）

  • SELECT *

    FROM table

    WHERE + 条件
  • <> 与 != 相等
  • WHERE .. BETWEEN .. AND ..
  • WHERE .. IS NULL 是否为空的字段
  • WHERE .. OR .. AND (AND的优先级高于OR,可以使用括号先处理OR)
  • 与OR相当但比OR作用更快 IN
    WHERE id IN (1002,1003,1004,1005)
  • NOT: 否定它之后所有跟的条件
    WHERE id NOT IN (1002,1003,1004,1005)

四 通配符

  • WHERE .. LIKE .. 操作符
  • ( % ) 表示任何字符出现任何次数

    LIKE ‘jet%’ , ‘%ailv%’ , ‘%lad’
  • ( _ ) 只匹配单个字符

    LIKE ‘_oney’ –> honey
  • 不要过度使用通配符

五 正则表达 REGEXP

  • WHERE .. REGEXP .. 类似LIKE但不需要通配符

  • WHERE name REGEXP ‘1000|2000’ 名字里有1000或2000

  • WHERE name REGEXP ‘[123] Ton ‘ 名字里是1 Ton 或者 2 Ton 或者 3 Ton 【另外】[1-9]Ton 1至9 Ton

  • 转义(escaping)

    WHERE name REGEXP '.000'
名字里有1000或2000等，.匹配任意字符
因此要搜索点，需要加上\\. 搜索-需要加上\\-

  • 定位符 ^ $ 

    REGEXP '^[0-9\\.]' 
^文本开始，表示文本以0-9的数字或.开头
