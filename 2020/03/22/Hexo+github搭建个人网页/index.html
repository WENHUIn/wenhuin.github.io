花了我一天的时间去学，别看教程简单，过程遇到的问题真是错综复杂毕竟咱也不是专门写代码的，随便一个就让人想放弃，还好大部分我都找到了解决方法，但仍然要继续进行探索。特此第一篇Post记录一下建站过程及看tutorial的笔记 :)

准备：vsc,github,有道云笔记 or 其他markdown的编写器

I 安装hexo （让我非常之无语的第一步）

(https://hexo.io/zh-cn/docs/）

直接按照这个步骤安装Node.js和Git。作为OS Catalina的用户非常之无语，因为这两个的安装包不是双击就能打开，一定要右键选择open with installation 才能安装。（劝退1）

接着是安装hexo,在终端terminal里运行。官方给的口令我现在都混乱了不知道是对还是错（而且切记不要复制$，那个是terminal自带的，只要复制后面的命令就可以了），因为我一安装就出现ERROR EACCES ,按照文档的方法也没用解决因为太难看懂了！！！于是我就跳过这个步骤了，直接用如下口令来验证是否安装了hexo。 (劝退2)

hexo -v
II 看Tutorial (非常好)

来自官方的教学视频，比文档读起来容易懂多了：

以下是从Tutorial 3 开始笔记：


1. Creating new site

hexo init <你自己创建的文件名>

并获得该提示：'INFO Start blogging with Hexo!'
cd <你自己创建的文件名>
hexo server

运行了hexo服务器并生成网站预览在 localhost:4000

2. Writing posts

hexo new <post name>
即可新建一个new post在_posts下
hexo new draft <draftpost name>
即可在source里新建一个draft文件夹并生成一个draft，但draft内容不会显示在网页上

hexo publish <draftpost name>
即可将draft显示在网页上
hexo new page <page name>
即可在source里新建一个page,和posts同级

3. Scaffolds

就是所有组成部分的template， 可以在对应的.md里修改模型。比如要修改post的样式，在post.md里即可进行修改。Customize your layout !!!

4. Tags & Categories

tags: [tag1,tag2,tag3]
在某个post.md里直接对Tag进行添加,并在主页显示tags的分类，点击任意一个即可进入该tag分类
categories:
- [catr1]
- [catr2]
- [catr3]
可以用来制作目录

5. Assets folder

在_config.yml下的writting里，修改post_asset_folder: true即可激活这个功能。

hexo new <post name>
直接创建一个post的文件在_post之下，可以拖动图片到该文件夹下

{% asset_img picname.jpg 自定义图片名字 %} 显示图片
{% asset_link picname.jpg %} 显示图片的link
{% asset_path picname.jpg %} 显示图片的路径

6. Theme

直接下载现成主题: https://hexo.io/themes/

  • 点击自己喜欢的主题进入到该github
  • 方法1：下载到themes文件夹并解压，然后删除前面的hexo-theme文字，只保留主题名
  • 方法2：在终端使用
git clone <复制的模板github路径>
缺点是目前我不知道咋放到themes下，都是再剪切过去
  • 修改击_config.yml里的theme为主题名即可

创建自己的主题:

如果创建一个theme,则需要包括【languages,layout,scripts,source】四个文件夹
